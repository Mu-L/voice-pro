Voice-Pro 4.0 업데이트 — 7개월 만에 성공한 uv 전환, 설치 시간 1시간에서 몇 분으로


안녕하세요, ABUS AI Korea입니다.

AI 더빙 스튜디오 Voice-Pro의 4.0 업데이트 소식을 전해드립니다. 이번 업데이트는 겉으로 보이는 새 기능보다, 프로그램의 뼈대를 통째로 갈아엎은 업데이트입니다. 결론부터 말씀드리면 이렇습니다.

■ 설치 시간: 1시간 이상 → 몇 분 (AI 모델 다운로드 제외)
■ 관리자 권한 없이 설치 가능 — 회사 PC에서도 OK
■ 최신 웹 UI 프레임워크(Gradio 6) 적용
■ 음성인식·음성합성 AI 엔진 전면 최신화 + 한국어 지원 CosyVoice3 추가

하나씩 자세히 설명드리겠습니다.


────────────────────────────
1. 7개월 전의 실패, 그리고 이번의 성공
────────────────────────────

사실 uv 전환은 이번이 첫 시도가 아닙니다. 약 7개월 전에도 Miniconda 기반 설치 방식을 uv로 바꾸려고 시도했다가 실패한 적이 있습니다.

당시 발목을 잡은 것은 pynini라는 라이브러리였습니다. Voice-Pro의 음성합성 엔진인 CosyVoice가 텍스트 정규화(숫자, 날짜 등을 읽는 법으로 바꾸는 작업)에 pynini를 사용했는데, 이 라이브러리는 Windows용 빌드가 아예 존재하지 않습니다. conda 생태계에서만 설치할 수 있었기 때문에, conda를 버리는 순간 CosyVoice가 함께 무너지는 구조였습니다.

이번에 전환에 성공한 결정적 계기는 CosyVoice 개발팀(FunAudioLLM)이 pynini를 버리고 wetext라는 순수 Python 라이브러리로 갈아탄 것입니다. Voice-Pro에 내장된 CosyVoice를 최신 버전으로 통째로 교체하면서 이 문제가 자연스럽게 해소되었고, 마침내 conda를 완전히 제거할 수 있었습니다.

7개월을 기다린 보람이 있었습니다.


────────────────────────────
2. 설치 속도 — "1시간 이상"에서 "몇 분"으로
────────────────────────────

기존 Voice-Pro는 Miniconda + pip 조합으로 설치했습니다. configure.bat 실행에만 1시간 이상, 첫 start.bat에서 또 1시간 이상이 걸렸습니다. 설치 도중 실패하면 처음부터 다시 시작해야 했고, 사용자마다 설치되는 패키지 버전이 미묘하게 달라져서 "제 PC에서만 안 돼요" 문제의 온상이 되기도 했습니다.

4.0부터는 Rust로 작성된 초고속 패키지 관리자 uv를 사용합니다.

• 파이썬 환경 구축이 몇 분 안에 끝납니다. 이제 설치에서 가장 오래 걸리는 건 파이썬이 아니라 AI 모델 다운로드(약 10GB)입니다.
• uv.lock 파일에 모든 패키지 버전이 고정되어 있어서, 누가 언제 설치해도 개발팀이 검증한 것과 100% 동일한 환경이 만들어집니다.
• 문제가 생기면 installer_files 폴더를 지우고 start.bat만 다시 실행하면 됩니다. 재설치가 몇 분이면 끝나기 때문에 부담이 없고, 다운로드해 둔 AI 모델(model 폴더)은 그대로 유지됩니다.
• uv, Python 3.12, 모든 패키지가 전부 installer_files 폴더 안에만 설치됩니다. 시스템을 건드리지 않는 완전한 포터블 방식이라, 삭제할 때도 폴더만 지우면 끝입니다.


────────────────────────────
3. 관리자 권한 없이 설치 — 회사 PC에서도 됩니다
────────────────────────────

이번 업데이트에서 개인적으로 가장 마음에 드는 부분입니다.

• configure.bat이 선택사항이 되었습니다. 관리자 권한이 없다면 그냥 건너뛰고 start.bat만 실행하세요. ffmpeg가 설치되어 있지 않으면 start.bat이 포터블 버전을 자동으로 받아서 씁니다.
• CUDA Toolkit과 Visual Studio Build Tools가 더 이상 필요 없습니다. PyTorch가 CUDA 런타임을 내장하고 있고, 모든 패키지가 미리 빌드된 상태로 설치되기 때문입니다. NVIDIA 그래픽 드라이버만 최신이면 됩니다.
• 제거(uninstall.bat)도 관리자 권한이 필요 없고, 강제 재부팅도 하지 않습니다.
• 보안 장비가 있는 회사 네트워크도 배려했습니다. 무료 번역 서비스가 차단·제한되는 환경에서는 자동으로 재시도하고, 그래도 안 되면 원문을 유지한 채 실패 건수를 알려줍니다. Azure 키가 있다면 .env 파일에 넣는 것만으로 Azure 번역/TTS로 전환할 수도 있습니다.

즉, 관리자 권한이 없는 사내 PC에서도 압축을 풀고 start.bat 한 번이면 설치가 됩니다.


────────────────────────────
4. Gradio 6 — 최신 웹 UI, 그리고 눈에 보이는 에러
────────────────────────────

웹 UI 프레임워크를 Gradio 6.20 최신 버전으로 올렸습니다.

• 모든 탭이 화면 전체 폭을 사용해서 훨씬 넓고 시원해졌습니다.
• 자막이 비디오 플레이어에 자막 트랙으로 바로 표시됩니다.
• 에러 표시 방식을 전면 개선했습니다. 기존에는 오류가 10초짜리 경고로 잠깐 떴다가 사라져서 놓치기 일쑤였는데, 이제는 닫기 전까지 화면에 남아 있는 빨간 에러 토스트로 표시되고, 원인(ffmpeg 없음, 미디어 미등록 등)을 알려주는 메시지가 함께 나옵니다.


────────────────────────────
5. AI 엔진 전면 최신화
────────────────────────────

핵심 엔진으로 사용되는 AI 모델·라이브러리를 모두 최신 버전으로 올렸습니다.

【런타임】
• Python 3.12, PyTorch 2.8.0 + CUDA 12.8 — RTX 50 시리즈 최신 그래픽카드를 지원합니다.

【음성인식(ASR)】
• faster-whisper 1.2.1 — large-v3-turbo, distil-large-v3.5 등 최신 모델 사용 가능
• openai-whisper 20250625, whisper-timestamped 1.15.9
• whisperX는 제거되었습니다. 오래된 의존성 고정이 Gradio 6 업그레이드를 막고 있었기 때문입니다. 기존에 whisperX를 쓰시던 설정은 자동으로 faster-whisper로 전환됩니다.

【음성합성(TTS) / 음성 클로닝】
• F5-TTS 1.1.21, kokoro 0.9.4, edge-tts 7.x
• CosyVoice를 업스트림 최신 버전으로 통째로 교체
• 신규: Fun-CosyVoice3-0.5B — 한국어를 포함한 9개 언어를 지원하는 최신 음성 클로닝 모델입니다. CosyVoice 탭에서 선택할 수 있고, 처음 사용할 때 공식 저장소에서 자동 다운로드됩니다. 한국어 음성 클로닝 품질이 궁금하셨던 분들은 꼭 써보세요.


────────────────────────────
6. 설치 방법 (요약)
────────────────────────────

1) GitHub에서 소스를 받습니다.
   https://github.com/abus-aikorea/voice-pro

2) start.bat 실행 — 이게 전부입니다.
   첫 실행 시 uv와 Python 3.12, 모든 패키지가 자동으로 설치되고(몇 분), 이어서 AI 모델(약 10GB)을 다운로드합니다.

3) 브라우저에서 Voice-Pro WebUI가 열립니다.

※ 관리자 권한이 있고 git/ffmpeg를 시스템에 설치하고 싶다면 configure.bat을 먼저 실행해도 됩니다(선택사항).
※ 업데이트는 update.bat, 제거는 uninstall.bat 또는 폴더 삭제로 끝납니다.


────────────────────────────
